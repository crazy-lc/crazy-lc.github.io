子集和问题（基于并行动态规划求解子集和问题）

一、问题描述

条件：给定集合 $S = {x_1,x_2,x_3,…,x_n}，x_i\in Z$ ，某整数定值 k

问题：判断集合是否存在 $S$ 的一个子集，使得其中的数加起来恰为目标值 k 。

具体求解的例子：给定一组整数，是否存在一个总和为零的非空子集？

例如：

给定 ${-7,-3,-2,5,8}$ ，答案是肯定的，因为子集 ${-3,-2,5}$ 总和为零。

二、常见解法

  • 回溯法

    类似 0-1背包问题。

    缺点：

    • 没有办法进行剪枝
    • 不适合进行并行优化

  • 动态规划

    为了计算 $S_n={x_1,x_2,x_3,…,x_n}$ 的所有子集的元素和；

    可以先计算 $S_{n-1}={x_1,x_2,x_3,…,x_{n-1}}$ 的所有子集的元素和。

    如果 $L$ 是一个由整数所构成的表，$x$ 是一个整数，我们用 $L+x$ 来表示

    通过对 $L$ 中每个元素增加 $x$ 而导出的整数列表。

    例如：如果 $L=<1,2,3,5,9>$ ，则 $L+2 = <3,4,5,7,11>$

    那么
    $$
    L_n 的所有子集和 = (L_{n-1} + x_n)\cup(L_{n-1})
    $$

    先求 $L_0 => L_1 => L_2 => … => L_{n-1} => L_n$

三、并行优化

假设条件：

假设有 n 个线程，S 的集合大小为 N，目标和为 k , $f(N)$ 为求 N 大小的集合的子集和问题的时间复杂度。
最差情况下，该问题的时间复杂度为：$f(N) = 2^N$

并行求解：

  1. 将 S 集合的 N 个整数分成 n 个互不相交的集合，标记为 $S_1$, $S_2$, $S_3$, …, $S_{n-1}$, $S_n$ . 将其分配给 n 个线程，通过动态规划算法分别进行求解。（注意空集的特殊处理）

    1）如果其中某个集合 $S_i$ 求得子集和 k， 则直接返回存在子集和 k，总的求解所需时间复杂度为 $f(N/n)$ 。

    2）若全都没有求得目标和，则每个线程返回相应的 子集和的集合 ，用 $X_i$ 用于表示第 i 个线程返回的集合。得到下面的矩阵 X 。

$$
X=\begin{pmatrix}
X_1\
X_2\
…\
X_n\
\end{pmatrix}，其中 X_i={x_1,x_2,…,x_{m_i}}，m_i 跟随 i 变化且m_i\le2^{N/n}
$$

  1. 若第一步中 n 个集合都未出现子集和 k。则进一步进行并行计算，将 $X_i$ 分为 n 段，每段由一个线程计算其与 $X_{i+1}$ 的笛卡尔积的和集合。

代码可产生的随机数的范围：

RAND_MAX: 2147483647

进一步优化措施：

  • 可求出集合 S 的所有正值之和 A，所有负值之和 B，则当某一子集和 s<0且-s>A 或 s>0且-s<B 时，该子集和可丢弃
  • 在第二步中，当归并到只剩下两个集合的子集和结果时，可使用Horowitz和Sahni于1974年提出的算法进行适当优化。

    时序图

    1 什么是时序图

    序列图（Sequence Diagram），又名时序图、循序图、顺序图，是一种UML交互图。它通过描述对象之间发送消息的时间顺序显示多个对象之间的动态协作。它可以表示用例的行为顺序，当执行一个用例行为时，其中的每条消息对应一个类操作或状态机中引起转换的触发事件。
    通俗的讲就是描述不同时间下各对象之间的消息或动作。

2 时序图基础

常规的时序图由以下几个部分组成：
角色
系统角色，可以是人、其他的系统或者子系统；
比如说，市民是一个角色，终端系统是一个角色
对象
对象是某一类实例，比方说张三，POS机；
对象包括三种命名方式：

  • 包括对象名和类名；
  • 只显示类名不显示对象名，即表示他是一个匿名对象；
  • 只显示对象名不显示类名。

生命线
一个对象的生命线就表示该对象的存活时间，是一条垂直的虚线，消息在虚线之间传递。
控制焦点
在生命线中的某一个点，对象的动作，也就是某一时刻对象做了什么。
消息
不同时间点上不同对象之间的消息交互。

甘特图

甘特图通过活动列表和时间刻度表示出特定项目的顺序与持续时间。甘特图中横轴表示时间，纵轴表示项目，条表表示期间计划和实际完成情况。直观表明计划何时进行，进展与要求的对比。便于管理者弄清项目的剩余任务，评估工作进度。

单例模式singleton

**要求:**定义一个类，只能由该类创建唯一的一个对象

  1. 该对象不能是栈/全局对象
  2. 只能将该对象放在堆上

应用场景:

  1. 可以用单例模式替换掉全局变量（减少全局变量便于管理）
  2. 读取配置文件中的信息
  3. 词典库，网页库
  4. 操作系统的文件系统
  5. 线程池一般也是单例模式
